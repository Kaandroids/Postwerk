# Veröffentlichen & Installieren

_Teilen Sie eine Automatisierung mit der Community oder installieren Sie eine und machen Sie sie zu Ihrer eigenen. Eine Installation funktioniert immer, weil beim Installieren alles, was die Automatisierung braucht, in Ihrem eigenen Konto neu erstellt wird._

## Installieren

Stöbern Sie im Marketplace, öffnen Sie ein Listing und klicken Sie auf **Installieren**. Beim Installieren wird die Automatisierung **tiefkopiert** und **jede referenzierte Ressource geklont** — Kategorien, Parametersets, Vorlagen, Webhook-Endpunkte und Wissensdatenbanken — in Ihr eigenes Konto. Da Ihnen die Kopie gehört, nutzen Läufe Ihr Postfach, Ihr Kontingent und Ihren SMTP, sodass die Ausführung einfach funktioniert.

Es gibt zwei Arten von Listings:

- **Öffentliches Listing** → Sie erhalten eine vollständig **bearbeitbare Kopie**, die im normalen Editor geöffnet wird. Passen Sie jeden Node an.
- **Privates Listing** → Sie erhalten eine **inhaltlich verborgene, gesperrte Kopie**. Alle Konstanten werden kopiert, damit die Logik läuft, aber Sie können die Interna nie einsehen. Sie konfigurieren nur die Werte, die der Autor als veröffentlichbar markiert hat, über die **Configure**-Oberfläche, binden dann ein Postfach an und aktivieren.

## Bezahlung

Die Preisgestaltung ist **reine Metadaten**. Ein Listing kann ein Preismodell und einen Preis angeben, und der Erwerb erzeugt eine Berechtigung — es wird jedoch nie etwas berechnet.

## Wissensdatenbank-Daten teilen

Wenn Sie eine Automatisierung veröffentlichen, die eine Wissensdatenbank referenziert, wählen Sie, wie viel davon mitreist:

| Richtlinie | Was übertragen wird |
|------------|---------------------|
| Nur Schema (Standard) | Das Schema der Wissensdatenbank; Käufer füllen oder importieren ihre eigenen Einträge |
| Vollständig | Schema plus Einträge, wortgetreu kopiert |
| Privates Listing | Wissensdatenbank ist verborgen — läuft, aber Einträge sind nicht lesbar |

> [!DANGER]
> Der Standard ist aus gutem Grund „Nur Schema". Wählen Sie **Vollständig** nur, wenn die Inhalte der Wissensdatenbank öffentlich sein sollen — niemals für proprietäre oder personenbezogene Daten.
